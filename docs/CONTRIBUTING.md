# Contributing to OpsHub

## Branching
- work from feature branches
- keep `main` stable
- merge through PRs

## Commits
Use conventional commits.

Examples:
- feat: add dashboard summary endpoint
- fix: handle empty lead states
- docs: update setup instructions
- chore: add lint config

## Rules
- small focused commits
- avoid mixing refactors with features
- document important changes in docs/
